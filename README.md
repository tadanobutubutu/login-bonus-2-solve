# Login Bonus 2 - Writeup

## 概要
`Login Bonus 2` は、バッファオーバーフローを利用してメモリ上のポインタを書き換え、情報を漏洩させる問題です。

## 脆弱性の詳細
- プログラム内で、ユーザー入力を `scanf("%[^\n]", password)` を用いて 100 バイトの固定長バッファ `password` に読み込んでいます。
- ここに 100 バイトを超える入力を与えることで、スタック上のデータを上書きするバッファオーバーフローが発生します。
- 認証に失敗した際、`debug_report(argv[0], "Auth NG");` と `debug_report(argv[0], "Invalid password: %s", password);` が呼び出されます。
- `debug_report` マクロは `printf("%s: " fmt "\n", progname, ##__VA_ARGS__)` に展開され、`argv[0]` が `progname` として渡されます。
- `argv` 配列（ポインタの配列）は本来スタックの高位に配置されていますが、このプログラムの配置ではバッファオーバーフローによって `argv[0]` の値を任意のアドレスに書き換えることが可能でした。
- `argv[0]` を `g_flag`（フラグが格納されているグローバル変数）のアドレスで上書きすることで、エラーメッセージのプログラム名表示部分にフラグを出力させることができます。

## 解決手法
1. `g_flag` のアドレス（`0x404040`）を特定します。
2. 104 バイトのパディングと、その後の `argv` 配列が配置されるであろう領域に `0x404040` を大量に送り込む（Spray）ペイロードを作成します。
3. ペイロードを送信すると、認証エラー時の `debug_report` の呼び出しにおいて、`argv[0]` が指す先がフラグ文字列となり、画面に表示されます。

**Flag**: `Alpaca{...}` (解決スクリプトの実行により取得可能)

この問題はgoogleのantigravityを使って解きました。
